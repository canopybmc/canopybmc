# Release Notes

This is `rc1` of the Canopy BMC release **2026.06**.

> [!NOTE]
> This release introduces breaking changes for HPE Proliant Gen11 platforms
> Users of previous versions (eg. _2026.04_) need to do a full update by using (`static.mtd.all.tar`)
>
> For users that want to use the BIOS (Next) boot order feature may have to load
> the BIOS defaults once to synchronise the BIOS variables correctly.

## Features
- OpenBMC
    - Upgrade to Yocto 6.0 (Wrynose) via OpenBMC rebase
    - OpenBMC bumped to `4cef1400fb` - 2026-07-06 ([#221](https://github.com/canopybmc/canopybmc/pull/221))
    - Weekly OpenBMC rebase job ([#177](https://github.com/canopybmc/canopybmc/pull/177))
- Build and release CI rework ([#70](https://github.com/canopybmc/canopybmc/issues/70), [#217](https://github.com/canopybmc/canopybmc/pull/217))
- Provide SBOM information per platform ([#176](https://github.com/canopybmc/canopybmc/pull/176))
- HPE Proliant Gen11
    - Enable Host BIOS update support ([#84](https://github.com/canopybmc/canopybmc/issues/84), [#210](https://github.com/canopybmc/canopybmc/pull/210))
    - Move to OpenBMC Linux kernel ([#178](https://github.com/canopybmc/canopybmc/pull/178))
    - Enable UBM devices ([#163](https://github.com/canopybmc/canopybmc/issues/163), [#162](https://github.com/canopybmc/canopybmc/pull/162))
    - bmcweb: increase http body limit ([#191](https://github.com/canopybmc/canopybmc/pull/191)
    - Restructure README.md and document OSFCI ([#185](https://github.com/canopybmc/canopybmc/pull/185))
    - Optimize BMC boot ([#73](https://github.com/canopybmc/canopybmc/pull/73))
    - Add IPMI test cases for HPE ProLiant Gen11 and implements OpenBMC ipmi fixes ([#195](https://github.com/canopybmc/canopybmc/pull/195))
    - Support for BIOS (Next) boot order and IP configuration ([#227](https://github.com/canopybmc/canopybmc/pull/227))
